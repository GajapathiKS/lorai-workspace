Free, private, local AI — chat, image gen, voice, RAG, agents — running on your machine at port 1842

1
Install Docker on your computer — macOS, Windows, or Linux docs.docker.com/get-docker → LorAI runs inside a Docker container — you only need Docker installed, nothing else to configure
2
Do you have Python 3.9 or newer installed?

Option 1 — Homebrew (recommended if you have it):

$ brew install python3

Option 2 — Official installer:

python.org/downloads/macos →

Verify: run python3 --version in Terminal — should print Python 3.x.x

Option 1 — Windows Package Manager:

> winget install Python.Python.3.12
Restart PowerShell after install so the PATH updates take effect.

Option 2 — Official installer (easiest for beginners):

python.org/downloads/windows →
During install: tick "Add python.exe to PATH" on the first screen — without this pip won't work.

Verify: open PowerShell, run python --version

Ubuntu / Debian:

$ sudo apt update && sudo apt install python3 python3-pip

Fedora / RHEL / CentOS:

$ sudo dnf install python3 python3-pip

Arch:

$ sudo pacman -S python python-pip

Verify: python3 --version && pip3 --version

Then, in your Terminal or PowerShell: $ pip install lorai-workspace Installs the Python client only — does not touch Docker, pull any image, or start anything
3
In a Python file or Python shell — on your computer: from lorai_workspace import LorAI; LorAI().chat("Hello") LorAI() is where Docker gets involved — it pulls gajapathiks/lorai-workspace:latest from Docker Hub (~2 GB) and starts the container automatically. This only happens once; subsequent runs skip straight to your code.

See it in action

Write Python on your machine. LorAI handles Docker, models, and everything else automatically.

quickstart.py Python — run on your computer 
from lorai_workspace import LorAI

# ← THIS line pulls Docker image + starts container
#   pip install does NOT touch Docker — only LorAI() does
ai = LorAI()

# Chat with a local LLM
print(ai.chat("What is the capital of France?"))

# Index your docs and ask questions (RAG)
ai.knowledge.ingest("/my/docs/")
print(ai.knowledge.ask("What is the refund policy?"))

# Generate an image (GPU required)
ai.image.generate(
    "a cat in a space suit",
    save_to="cat.png"
)

# Transcribe audio to text
text = ai.voice.transcribe("meeting.mp3")
print(text)

What's included

Ready

Chat & LLMs

Local chat, text completion, and embeddings via Ollama. Streaming supported. Default model: phi3:mini (works on CPU).

Ready

Model Hub

Pull any Ollama model — llama3, mistral, codestral, gemma. Benchmark, list, and remove models via ai.hub.

GPU

Image Generation

SDXL Turbo text-to-image and image editing via ai.image. Requires NVIDIA GPU 8GB+ VRAM.

GPU

Video Generation

CogVideoX text-to-video via ai.video. Requires NVIDIA GPU 12GB+ VRAM.

Ready

Voice — STT & TTS

Whisper.cpp speech-to-text and Piper text-to-speech via ai.voice. 6 voice options.

Ready

RAG / Knowledge Base

ChromaDB-backed document ingestion, semantic search, and RAG Q&A via ai.knowledge.

Ready

Agents

ReAct agent loop with tools: search, file read/write, code run, knowledge. Via ai.agents.

Ready

Code Execution

Generate and run Python, Bash, JavaScript in a sandboxed environment via ai.code.

Ready

Vision & OCR

Analyse images and extract text using LLaVA multimodal models via ai.vision.

Experimental

LoRA Adapters

Load and apply GGUF LoRA fine-tunes on top of any base model via ai.lora.

GPU

Music Generation

MusicGen text-to-music. Requires NVIDIA GPU 4GB+ VRAM.

Ready

Browser Desktop

Full Linux desktop accessible in your browser via noVNC at port 6080.

CLI Quick Reference

terminal — run on your computer Shell 
lorai-workspace start               # pull image + start Docker container
lorai-workspace start --gpu         # with GPU passthrough (for image/video/music)
lorai-workspace stop                # stop the container
lorai-workspace status              # health, loaded models, GPU/CPU info
lorai-workspace chat "Hello"        # chat from terminal
lorai-workspace pull llama3         # download a model
lorai-workspace desktop             # open browser desktop (noVNC)
lorai-workspace logs                # tail container logs
lorai-workspace version             # show version and port
Using Node.js, Go, Rust, Java, C#, Ruby, PHP, or cURL? — LorAI Workspace exposes a standard HTTP API on localhost:1842/v1. Any existing OpenAI-compatible client library works by pointing its base_url there.  See examples for all 10 languages →

Beta Release

v0.1.0-beta — Core API is stable. Some features are still being refined. We're looking for early adopters to test and give feedback.  View changelog →
  • 27 API endpoints — chat, images, audio, embeddings, RAG, agents, code, vision, LoRA, model hub
  • Python client with auto Docker management, 9 service classes, and CLI
  • CPU + GPU support — chat/voice/RAG run on CPU; image/video/music need NVIDIA GPU
  • First startup downloads ~4 GB total (Docker image + phi3:mini) — subsequent runs are instant
  • No authentication — designed for local development, not internet-facing deployment
